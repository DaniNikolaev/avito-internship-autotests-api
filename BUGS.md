## BUG-001: Статистика доступна после удаления объявления

### Описание
После удаления объявления через DELETE /api/2/item/{id}, GET запрос к эндпоинтам статистики (/api/1/statistic/{id} и /api/2/statistic/{id}) возвращает 200 OK и статистику удаленного объявления.

### Ожидаемое поведение
Следуя принципам REST API, статистика не должна существовать без объявления. GET запрос должен возвращать 404 Not Found.

### Фактическое поведение
- **Статус**: 200 OK
- **Тело ответа**: статистика удаленного объявления

### Затронутые эндпоинты
- `GET /api/1/statistic/{id}`
- `GET /api/2/statistic/{id}`

### Приоритет
Высокий — нарушение REST принципов, может привести к некорректной работе клиентов

### Тесты с xfail
- `test_get_statistic_v1_after_delete_advertisement`
- `test_get_statistic_v2_after_delete_advertisement`

## BUG-002: Статистика v2 возвращает 404 вместо 400 при некорректном ID

### Описание
При запросе статистики v2 с некорректным ID (не UUID), API возвращает 404 Not Found вместо 400 Bad Request.

### Ожидаемое поведение
При передаче некорректного формата идентификатора API должен возвращать 400 Bad Request с информативным сообщением.

### Фактическое поведение
- **Статус**: 404 Not Found
- **Тело ответа**: `{"result": "statistic not found", "status": "400"}`

### Затронутые эндпоинты
- `GET /api/2/statistic/{id}`

### Приоритет
Средний — некорректный HTTP статус, но пользователь получает информацию о том, что ресурс не найден

### Тесты с xfail
- `test_get_statistic_v2_with_incorrect_advertisement_id`

## BUG-003: Создание объявления с отрицательными значениями возвращает 200

### Описание
При создании объявления через POST /api/1/item с отрицательными значениями в полях (sellerID, price, statistics.likes, statistics.viewCount, statistics.contacts) API возвращает 200 OK и создает объявление.

### Ожидаемое поведение
Отрицательные значения не должны приниматься. API должен возвращать 400 Bad Request с сообщением о некорректных данных.

### Фактическое поведение
- **Статус**: 200 OK
- **Создается объявление** с отрицательными значениями

### Затронутые поля
- sellerID
- price
- statistics.likes
- statistics.viewCount
- statistics.contacts

### Приоритет
Средний — валидация не работает, могут накапливаться некорректные данные

### Тесты с xfail
- `test_create_advertisement_v1_with_negative_int_fields`

## BUG-004: DELETE несуществующего объявления возвращает некорректный формат ошибки

### Описание
При удалении несуществующего объявления через DELETE /api/2/item/{id} API возвращает 404 Not Found, но тело ответа содержит статус 500.

### Ожидаемое поведение
API должен возвращать 404 Not Found с телом, содержащим информацию о том, что ресурс не найден.

### Фактическое поведение
- **Статус**: 404 Not Found
- **Тело ответа**: `{"result": {"message": "", "messages": null}, "status": "500"}`

### Затронутые эндпоинты
- `DELETE /api/2/item/{id}`

### Приоритет
Низкий — статус 404 корректный, проблема только в теле ответа

### Тесты с xfail
- `test_delete_non_exist_advertisement_v2`

- **Тело ответа**: пустое или `{"result": {"message": "", "messages": {}}, "status": "не передан объект - объявление"}`

### Затронутые поля
- Все обязательные поля (sellerID, name, price, statistics)

### Приоритет
Низкий — статус 400 корректный, проблема только в информативности ответа

### Тесты с xfail
- `test_create_advertisement_v1_with_empty_fields`
